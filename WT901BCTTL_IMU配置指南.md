# WT901BCTTL IMU配置指南

## 📌 产品信息

- **型号**: WT901BCTTL
- **厂商**: 维特智能 (WitMotion)
- **类型**: 9轴姿态传感器 (AHRS/IMU)
- **传感器**: 3轴加速度 + 3轴陀螺仪 + 3轴磁力计
- **接口**: TTL串口 (UART)
- **特点**: 内置卡尔曼滤波，直接输出欧拉角

---

## 🔌 硬件连接

### 接线方式

| WT901BCTTL引脚 | STM32引脚 | 说明 |
|---------------|-----------|------|
| VCC (红)      | 3.3V/5V   | 电源（支持3.3V和5V） |
| GND (黑)      | GND       | 地线 |
| TX (绿)       | USART_RX  | IMU发送 → STM32接收 |
| RX (白)       | USART_TX  | IMU接收 ← STM32发送（可不接）|

### 示例连接

```
WT901BCTTL       STM32F407
-----------     -----------
VCC (红)   -->   5V
GND (黑)   -->   GND
TX  (绿)   -->   PA3 (USART2_RX)
RX  (白)   -->   PA2 (USART2_TX) [可选]
```

**⚠️ 注意**：
- WT901BCTTL的TX连接到STM32的RX
- WT901BCTTL的RX连接到STM32的TX（如果需要配置IMU）
- 如果只接收数据，RX引脚可以不接

## ⚙️ STM32CubeMX配置

### 1. 配置USART（用于IMU）

打开STM32CubeMX，配置一个USART用于连接IMU：

#### 推荐配置：USART2

1. **模式**: Asynchronous（异步模式）
2. **波特率**: 115200 bps（WT901BCTTL默认波特率）
3. **数据位**: 8 Bits
4. **校验位**: None
5. **停止位**: 1
6. **硬件流控**: Disable

#### 中断配置

- 在 **NVIC Settings** 中启用 `USART2 global interrupt`
- 优先级可设为中等（如 Preemption Priority = 5）

### 2. 生成代码

配置完成后：
1. 点击 **GENERATE CODE**
2. 生成的代码会自动包含USART初始化

## 💻 代码配置

### 步骤1：设置IMU串口

在 `main.c` 的配置区（约第90-91行）修改：

```c
#define USE_IMU          1        // 1=使用IMU, 0=不使用IMU
#define IMU_UART         huart2   // 修改为您的USART句柄
```

**常用串口句柄**：
- `huart1` - USART1
- `huart2` - USART2
- `huart3` - USART3
- `huart6` - USART6

### 步骤2：确认串口波特率

在 CubeMX 中确认 USART 波特率为 **115200**

如果需要修改WT901BCTTL的波特率，请参考维特智能官方工具。

## 📡 数据格式说明

### WT901BCTTL数据帧结构

每帧11字节：

| 字节 | 说明 |
|------|------|
| 0 | 帧头：0x55 |
| 1 | 数据类型 |
| 2-3 | 数据1（低字节+高字节） |
| 4-5 | 数据2（低字节+高字节） |
| 6-7 | 数据3（低字节+高字节） |
| 8-9 | 温度/其他 |
| 10 | 校验和（前10字节之和） |

### 数据类型

| 类型 | 说明 | 数据内容 |
|------|------|---------|
| 0x51 | 加速度 | ax, ay, az |
| 0x52 | 角速度 | gx, gy, gz |
| 0x53 | 角度 | roll, pitch, yaw（**最重要**）|
| 0x54 | 磁场 | mx, my, mz |

### 数据解析示例

**角度帧**（0x53）：
```
55 53 E2 D0 65 01 1F 02 00 00 2F
```

解析：
- `55` - 帧头
- `53` - 角度数据
- `E2 D0` - Roll = -12062（int16）
- `65 01` - Pitch = 357（int16）
- `1F 02` - Yaw = 543（int16）
- `2F` - 校验和

转换为角度：
```c
roll = -12062 / 32768.0 * 180.0 = -66.3°
pitch = 357 / 32768.0 * 180.0 = 2.0°
yaw = 543 / 32768.0 * 180.0 = 3.0°
```

## 🎯 IMU在里程计中的作用

### 为什么要用IMU？

**仅用编码器的问题**：
- ❌ 轮子打滑 → 角度累积误差
- ❌ 轮径不准 → 转向误差
- ❌ 地面不平 → 长时间漂移

**加上IMU的优势**：
- ✅ 角度数据更精确（卡尔曼滤波）
- ✅ 不受轮子打滑影响
- ✅ 长时间运行误差小

### 数据融合策略

在 `USE_IMU = 1` 模式下：
- **角度（th）**：100% 使用 IMU 的 yaw 角
- **角速度（wz）**：100% 使用 IMU 的 gyro_z
- **位置（x, y）**：使用编码器位移 + IMU 角度
- **线速度（vx, vy）**：使用编码器计算

**结果**：编码器提供位移，IMU提供方向，精度大幅提升！

## ⚠️ 常见问题

### 问题1：收不到IMU数据

**现象**：`th` 和 `wz` 始终为 0

**排查**：
1. ✅ 检查接线（TX ↔ RX）
2. ✅ 检查波特率（115200）
3. ✅ 检查串口中断是否启用
4. ✅ 检查 `IMU_UART` 宏定义是否正确
5. ✅ 用示波器检查IMU的TX引脚是否有数据

**调试代码**：
```c
// 在HAL_UART_RxCpltCallback中添加
if(huart->Instance == IMU_UART.Instance) {
    // 翻转LED，确认进入中断
    HAL_GPIO_TogglePin(GPIOD, GPIO_PIN_12);
}
```

### 问题2：角度数据跳变

**现象**：`th` 突然从 3.14 跳到 -3.14

**原因**：角度归一化到 [-π, π]

**解决**：这是正常现象，ROS端会处理

### 问题3：角度方向相反

**现象**：顺时针转，`th` 却增大

**原因**：IMU安装方向不对

**解决**：
```c
// 在Parse_IMU_Data函数中，yaw取反
imu_data.yaw = -yaw / 32768.0f * PI;
```

### 问题4：数据不稳定

**现象**：`th` 抖动很大

**排查**：
1. 检查IMU供电是否稳定
2. 检查附近是否有强磁场干扰（磁力计）
3. 尝试远离电机和PWM信号线

## 🔧 高级配置

### 修改WT901BCTTL设置

维特智能提供了上位机软件可以配置：
- 波特率（9600/115200/460800等）
- 输出频率（0.1Hz~200Hz）
- 静止省电模式
- 磁场校准

**下载地址**：
- 官网：https://wit-motion.com
- 搜索：WT901BCTTL上位机

### 关闭不需要的数据帧

如果只需要角度数据，可以关闭加速度、角速度帧，减少数据量：
- 打开上位机
- 只勾选"角度输出"
- 关闭其他输出

**优点**：减少串口数据量，降低MCU负载

## 📊 性能优化

### 输出频率建议

| 应用场景 | IMU频率 | 里程计频率 |
|---------|---------|-----------|
| 低速机器人 | 50Hz | 10Hz |
| 中速机器人 | 100Hz | 20Hz |
| 高速机器人 | 200Hz | 50Hz |

**配置方法**：
1. IMU频率：用上位机设置
2. 里程计频率：修改 `HAL_Delay(100)` 和 `DT`

### 降低CPU占用

如果CPU占用高，可以：
1. 降低IMU输出频率
2. 只接收角度帧（0x53）
3. 使用DMA接收串口数据
