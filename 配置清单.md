# 机器人参数配置清单

## 🔧 必须配置的参数（main.c 第64-91行）

### ✅ 步骤1：选择机器人类型
```c
#define ROBOT_TYPE  0  // 改为您的机器人类型
```
- [ ] `0` = 差速驱动（两轮）
- [ ] `1` = 麦克纳姆轮（四轮）
- [ ] `2` = 全向轮

---

### ✅ 步骤6：配置IMU（可选但强烈推荐）

如果您有 **WT901BCTTL IMU**，可以大幅提高角度精度！

```c
#define USE_IMU          1        // 1=使用IMU, 0=不使用
#define IMU_UART         huart2   // 改为您的USART
```

#### 硬件连接：
| WT901BCTTL | STM32 |
|-----------|-------|
| VCC (红) | 5V |
| GND (黑) | GND |
| TX (绿) | USART_RX (如PA3) |

#### CubeMX配置：
1. 配置一个USART（如USART2）
2. 波特率：**115200**
3. 启用 USART 中断

## 🧪 验证步骤

### 1. 编译上传
- [ ] 代码编译成功
- [ ] 程序下载成功

### 2. 串口测试
- [ ] 串口助手连接成功
- [ ] 能看到格式为 `$x,y,th,vx,vy,wz#` 的数据

### 3. 前进测试
- [ ] 手动推动机器人前进 **1米**
- [ ] 观察 `x` 值是否接近 `1.000`
- [ ] 如果差距大，调整 `WHEEL_DIAMETER` 或 `ENCODER_PPR` 或 `GEAR_RATIO`

### 4. 旋转测试
- [ ] 原地旋转机器人 **360度**
- [ ] 观察 `th` 值是否接近 `6.283` (2π)
- [ ] 如果差距大：
  - 有IMU：检查IMU连接
  - 无IMU：调整 `WHEEL_BASE`

### 5. IMU测试（如果启用）
- [ ] 旋转机器人，`th` 实时变化
- [ ] 静止时，`th` 稳定不漂移
- [ ] 有IMU时角度精度明显优于纯编码器

## ⚠️ 常见错误

| 现象 | 原因 | 解决 |
|------|------|------|
| x值太大 | 轮子直径设置过大 | 减小WHEEL_DIAMETER |
| x值太小 | 轮子直径设置过小 | 增大WHEEL_DIAMETER |
| 前进但x为负 | 编码器方向反了 | 加负号：`-__HAL_TIM_GET_COUNTER` |
| th值偏差大 | 轮距测量不准 | 重新测量WHEEL_BASE |
| 数据全为0 | 未读取编码器 | 检查定时器配置 |
| th不变化(有IMU) | IMU未连接或配置错误 | 检查接线和串口配置 |
| th方向反(有IMU) | IMU安装方向相反 | yaw取反或物理翻转IMU |
